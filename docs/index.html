NMDK

The Nostr Markets Development Kit is a curated fork of Nostr client packages, protocol drafts, demo clients, and payment drivers that add marketplace functionality without locking applications to one payment method.

Install TypeScript Install Dart NDK

What this is

Marketplace Nostr, not a payment silo.

NMDK keeps the marketplace layer focused on listings, order lifecycle events, escrow policy discovery, payment proofs, recovery, and settlement state. Cashu, EVM, and future payment rails plug in as drivers.

TypeScript

Install the marketplace fork of nostr-tools.

Use the fork when you want marketplace event helpers, order state grouping, escrow policy routing, and payment-policy integration in browser or Node clients.

Core marketplace runtime

npm install nostr-tools@git+https://github.com/sudonym-btc/nostr-tools.git#marketplace

Import marketplace helpers from nostr-tools/marketplace.

Add Cashu escrow driver

npm install @sudonym-btc/marketplace-cashu@git+https://github.com/sudonym-btc/marketplace-cashu-ts.git#main

Cashu support mints P2PK-locked escrow proofs after a buyer pays a mint quote.

Add EVM escrow driver

npm install @sudonym-btc/marketplace-evm@git+https://github.com/sudonym-btc/marketplace-evm-ts.git#main \
  @sudonym-btc/marketplace-evm-contracts@git+https://github.com/sudonym-btc/marketplace-evm-contracts.git#main

EVM support handles contract calls, validation, account abstraction, and swap-aware settlement flows.

Dart

Install the NDK fork for Flutter or Dart clients.

The Dart NDK fork is included in NMDK, but marketplace-specific functionality has not landed there yet. The package is pinned so Dart clients can track the same Nostr base as the TypeScript marketplace work.

Dart NDK fork

dart pub add \
  "ndk:{git:{url: https://github.com/sudonym-btc/ndk.git, ref: marketplace, path: packages/ndk}}"

Flutter projects can use the same command through flutter pub add with the same dependency descriptor.

Driver add command shape

# Planned once Dart marketplace drivers exist:
dart pub add \
  "nmdk_cashu_driver:{git:{url: <cashu-driver-repo>}}"
dart pub add \
  "nmdk_evm_driver:{git:{url: <evm-driver-repo>}}"

Dart drivers should mirror the TypeScript model: optional packages that implement marketplace payment policies without owning Nostr relay IO.

Payment model

Payment-method agnostic, Lightning-connected.

Marketplace events describe intent and verifiable state. Drivers decide how funds are locked, verified, recovered, released, or refunded. That keeps the protocol open to Cashu, EVM, and future payment methods.

Lightning remains the practical edge of the system: buyers can enter through invoices, mints, or swap providers, and sellers can exit back through Lightning when they want spendable settlement.
